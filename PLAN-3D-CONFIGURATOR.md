# 🏗️ План реализации 3D-конфигуратора зданий

**Референс:** https://evrazsteelbox.ru/configurator-3d/calc_page/
**Дата:** 1 июля 2026

---

## 📦 Что умеет референс-конфигуратор

- **3D-визуализация** здания в реальном времени (вращение, масштаб)
- **5 серий каркасов** под разные нагрузки и назначения
- **Климатические зоны:** снег, ветер, сейсмика (справочник городов РФ)
- **Крановое оборудование:** 3.2 / 5 / 10 тонн, 1 или 2 крана, подкрановые пути
- **Стены:** сэндвич-панели / профлист, толщина, горизонтальная/вертикальная раскладка
- **Кровля:** сэндвич / мембрана / профлист, выбор толщины
- **18 цветов RAL** для панелей и нащельников
- **Элементы заполнения:** окна, ленточное остекление, двери, ворота
- **Цоколь и парапет** с выбором высоты
- **СМР:** фундамент, ж/б плита пола, доставка, монтаж каркаса, монтаж ограждений
- **PDF-коммерческое предложение** за 2–3 минуты
- **Шаг 10 см** по всем габаритам (Ш×Д×В)

## 📅 План реализации (14 недель)

### Этап 1: 3D-ядро — недели 1–5
```
Неделя 1–2: Архитектура и прототип
- Выбор стека: Three.js + React Three Fiber
- Подготовка GLTF/GLB моделей для каждой серии каркаса
- Базовый рендеринг: стены, крыша, пол
- Орбитальное управление камерой (rotate, zoom, pan)

Неделя 3–4: Каркас и элементы
- Отрисовка колонн, ферм, прогонов, связей по серии каркаса
- Переключение между 5 сериями в реальном времени
- Подсветка элементов, легенда

Неделя 5: Материалы и цвета
- Текстуры сэндвич-панелей (горизонтальная/вертикальная раскладка)
- 18 цветов RAL — запекание в текстуру
- Профлист, мембранная кровля
```

### Этап 2: Расчётный движок — недели 1–4 (параллельно с 3D)
```
Неделя 1–2: База данных и формулы
- Сортамент металла (двутавр, швеллер, уголок, труба)
- Таблицы нагрузок по СП 20.13330 (снег, ветер по районам)
- Расчёт металлоёмкости по серии и габаритам

Неделя 3–4: Сложные расчёты
- Крановые нагрузки (3.2/5/10т, 1 или 2 крана)
- Подкрановые пути: балки, рельсы, упоры
- Сейсмичность (по СП 14.13330)
- Фундамент и ж/б плита пола (предварительный расчёт, точный — после геологии)
```

### Этап 3: UI/UX — недели 5–8
```
Неделя 5–6: Визард (7 шагов)
- Шаг 1: Назначение здания → выбор серии каркаса
- Шаг 2: Габариты (Ш×Д×В, шаг 10 см, ползунки + поля ввода)
- Шаг 3: Климатическая зона (город → снеговой/ветровой район)
- Шаг 4: Стены и кровля (тип, толщина, цвет)

Неделя 7–8: Завершение визарда
- Шаг 5: Краны и оборудование
- Шаг 6: Окна, ворота, двери (drag-and-drop на 3D-модель)
- Шаг 7: СМР и контакты для КП
- Адаптивная вёрстка (десктоп + планшет, мобильная версия с оптимизированным 3D)
```

### Этап 4: Интеграции — недели 9–10
```
Неделя 9: PDF и КП
- Puppeteer/Playwright → рендер страницы в PDF
- Шаблон КП с логотипом, ценами, 3D-рендером, спецификацией
- Отправка на email

Неделя 10: CRM
- Вебхук в CRM (amoCRM / Битрикс24 / своя)
- Сохранение конфигурации (восстановление по ссылке)
- История расчётов в личном кабинете
```

### Этап 5: Тестирование и запуск — недели 11–14
```
Неделя 11: Функциональное тестирование
- Все комбинации: 5 серий × габариты × климат × краны × цвета
- Граничные значения (мин/макс габариты для каждой серии)
- Валидация цен с инженерным Excel

Неделя 12: Производительность и мобильные
- Оптимизация 3D (LOD, frustum culling, сжатие текстур)
- Мобильная версия (оптимизированная 3D или статичные ракурсы)
- Lighthouse: Target > 60 FPS на среднем десктопе

Неделя 13: UAT и правки
- Тестирование с реальными менеджерами
- Сбор обратной связи, правки UI
- A/B-тест старой и новой страницы калькулятора

Неделя 14: Запуск
- Деплой на прод
- Обучение менеджеров
- Мониторинг ошибок (Sentry)
```

# 📞 План реализации AI-прозвона холодных клиентов

**Назначение:** автоматический обзвон базы, квалификация лидов по статусу, передача горячих в CRM

**Цена прозвона:** 20–25 ₽/минута (за отвеченный звонок)

---

## 💰 Бюджет

| Компонент | Стек | Время | Стоимость |
|---|---|---|---|
| Телефония (VoIP/SIP) | Exolve / VoxImplant / Twilio, SIP-транк | 2–3 недели | 150 000 – 200 000 ₽ |
| Распознавание речи (STT) | Yandex SpeechKit / Tinkoff VoiceKit / Whisper | 2 недели | 100 000 – 150 000 ₽ |
| Синтез речи (TTS) | ElevenLabs / Yandex SpeechKit / SaluteSpeech | 1 неделя | 80 000 – 120 000 ₽ |
| Разговорный движок (LLM) | DeepSeek / GPT-4o, промпт-инжиниринг, сценарии | 2–3 недели | 150 000 – 250 000 ₽ |
| Классификация лидов | NLP + правила, статусы (горячий/тёплый/холодный/не целевой) | 1–2 недели | 80 000 – 120 000 ₽ |
| CRM-интеграция | Webhook → amoCRM / Bitrix24, карточка лида | 1 неделя | 60 000 – 80 000 ₽ |
| Дашборд и мониторинг | Статистика звонков, записи, конверсия | 2 недели | 100 000 – 150 000 ₽ |
| Тестирование и тюнинг | A/B-тесты скриптов, дообучение классификатора | 2 недели | 100 000 – 150 000 ₽ |
| **ИТОГО** | | **13–16 недель (3–4 месяца)** | **820 000 – 1 220 000 ₽** |

---

## 📅 План реализации (13–16 недель)

### Этап 1: Телефония — недели 1–3

- [ ] Выбор провайдера телефонии (Exolve/МТТ — российские, Twilio — международный)
- [ ] Аренда номеров (пул номеров, ротация для избежания блокировок)
- [ ] Настройка SIP-транка или WebRTC
- [ ] Исходящие звонки: дозвон, определение автоответчика, переключение на оператора
- [ ] Запись разговоров (WAV/MP3, хранение)
- [ ] Обработка состояний: не ответил, занято, автоответчик, сброс

### Этап 2: Распознавание и синтез речи — недели 3–5

- [ ] STT: потоковое распознавание (Yandex SpeechKit — лучший для русского)
- [ ] Обработка пауз, перебиваний, неразборчивой речи
- [ ] TTS: генерация голоса (ElevenLabs для премиум, SaluteSpeech для эконома)
- [ ] Базовые голосовые модели: мужской/женский, темп речи, интонации пауз
- [ ] Потоковая передача аудио: телефон ↔ STT ↔ LLM ↔ TTS ↔ телефон

### Этап 3: Разговорный движок — недели 5–8

- [ ] Проектирование скриптов продаж (открытие → прогрев → предложение → возражения → закрытие)
- [ ] Системный промпт LLM: роль, контекст компании, продукт, границы
- [ ] Обработка возражений: «дорого», «у нас уже есть», «перезвоните позже», «не интересует»
- [ ] Сбор информации: имя, должность, потребность, бюджет, сроки, ЛПР
- [ ] Эскалация на живого менеджера при триггерах (клиент просит соединить, сложное возражение)
- [ ] Ограничения: не врать, не давать несуществующих скидок, соблюдать 152-ФЗ

### Этап 4: Классификация лидов — недели 8–10

Статусы после разговора:

| Статус | Критерии | Действие |
|---|---|---|
| 🔴 **Не целевой** | Не профиль, ошиблись номером, физлицо | Удалить из базы |
| ⚫ **Не дозвон** | Не ответил (3 попытки), автоответчик, занято | Перенести на следующую волну |
| 🔵 **Холодный** | «Не интересует», «нет потребности» | Оставить в базе, прогрев через 3 мес |
| 🟡 **Тёплый** | Интерес, но «перезвоните через месяц», «пришлите КП» | Поставить задачу менеджеру |
| 🟢 **Горячий** | Готов к встрече/расчёту, назвал бюджет/сроки | Мгновенно в CRM → звонок менеджера |

Классификация:
- [ ] Правила на основе ключевых фраз и тональности
- [ ] LLM-классификатор (отдельный прогон всего диалога → JSON со статусом и причинами)
- [ ] Извлечение сущностей: имя, компания, телефон, email, потребность, бюджет

### Этап 5: CRM и дашборд — недели 10–12

- [ ] Webhook в CRM: создание/обновление карточки лида с полями
- [ ] Поля: статус, источник, дата звонка, запись, транскрипция, теги
- [ ] Дашборд для руководителя:
  - Звонков сегодня / дозвон / горячих / конверсия
  - График по дням и часам (лучшее время для прозвона)
  - Слушать записи, читать транскрипции
  - Сравнение скриптов (A/B)
- [ ] Уведомления в Telegram: «🔥 Горячий лид: Иван, СтройПроект, склад 1000 м²»

### Этап 6: Тестирование и запуск — недели 12–16

- [ ] Тестовый прозвон на 100 номерах (внутренняя БД)
- [ ] Анализ: % дозвона, средняя длительность, конверсия в горячих
- [ ] Донастройка скриптов и промптов по результатам
- [ ] A/B-тест двух скриптов
- [ ] Обучение менеджеров работе с горячими лидами
- [ ] Запуск на боевой базе (500–1000 номеров)
