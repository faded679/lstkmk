## 1. 🎙️ Голосовой AI-помощник на сайте

> **Цель:** AI-консультант не только пишет, но и **говорит** — полноценный голосовой диалог с посетителем сайта.

### Текущее состояние
- ✅ Голосовой **ввод** (STT) — работает в Calculator и AiAssistant (Web Speech API)
- ✅ Текстовый AI-чат — отвечает через DeepSeek API
- ❌ Голосового **вывода** (TTS) нет — только текст

### План реализации

#### Этап 1. Выбор TTS-движка (1 день)
- [ ] Оценить Web Speech API `SpeechSynthesis` (бесплатно, встроено в браузер)
- [ ] Сравнить с ElevenLabs API (качество выше, платно)
- [ ] Решение: стартуем с Web Speech API (бесплатно, 95% браузеров)
- [ ] Для премиум-качества добавить ElevenLabs как fallback

#### Этап 2. Озвучка ответов AI (3 дня)
- [ ] Модифицировать `AiAssistant.tsx` — добавить кнопку 🔊 и автоозвучку
- [ ] Разбивка длинных ответов на фразы (по предложениям/абзацам)
- [ ] Визуальный индикатор «говорит» (анимированный эквалайзер / пульсация)
- [ ] Очередь сообщений — не перебивать текущий ответ
- [ ] Управление: громкость,暂停/продолжить, перемотка

#### Этап 3. Голос в калькуляторе (2 дня)
- [ ] Озвучка результатов расчёта («Ваш склад 1440 м² будет стоить 6 693 000 ₽»)
- [ ] Проговаривание пояснений при смене параметров
- [ ] Кнопка «Слушать» рядом с итоговой ценой

#### Этап 4. Анимация «говорящего ассистента» (2 дня)
- [ ] SVG-аватар (рот синхронизирован с аудиопотоком)
- [ ] Состояния: idle / слушает / думает / говорит / ошибка
- [ ] CSS-анимации перехода между состояниями

#### Этап 5. Тестирование и фиксы (2 дня)
- [ ] Кросс-браузерное тестирование (Chrome, Firefox, Safari, мобильные)
- [ ] Обработка ошибок (нет поддержки Speech, нет микрофона, таймауты)
- [ ] Измерение latency (задержка между запросом и началом озвучки)

### Технический стек
```
TTS:        Web Speech API (primary) + ElevenLabs API (optional premium)
STT:        Web Speech API (уже есть)
Анимация:   CSS @keyframes + requestAnimationFrame
Состояния:  useReducer (idle | listening | thinking | speaking | error)
```

### Результат
Пользователь заходит на сайт → жмёт 🎤 → задаёт вопрос голосом → AI отвечает текстом **и голосом** через колонки/наушники. Полноценный голосовой диалог.

## 2. 🏗️ Детализированная схема здания

> **Цель:** В калькуляторе — не просто коробка, а **реалистичная модель** с колоннами, фермами, прогонами, связями и раскосами.

### Текущее состояние
- ✅ Изометрическая 3D-коробка (стены + крыша)
- ✅ Вентиляционный конёк для сельхоза
- ✅ Размеры (ширина × длина × высота)
- ❌ Нет колонн, ферм, прогонов, связей

### План реализации

#### Этап 1. Проектирование структуры (1 день)
- [ ] Изучить реальные схемы каркасов из Excel-калькулятора
- [ ] Определить правила отрисовки для каждого типа здания:
  - **Склад/производство:** колонны через 6м, двутавровые фермы, прогоны, связи
  - **Сельхоз:** ПСК-фермы, колонны через 4.8м, раскосы
  - **Автосервис/гараж:** облегчённый каркас, колонны через 6м
  - **До 300 м²:** упрощённый каркас

#### Этап 2. Фермы и колонны (3 дня)
- [ ] Отрисовка колонн (вертикальные линии с утолщением на торцах)
- [ ] Отрисовка ферм (стропильные, треугольный профиль)
- [ ] Расстановка с шагом согласно типу здания
- [ ] Цветовое кодирование: колонны = тёмно-синий, фермы = красный

#### Этап 3. Прогоны, связи, раскосы (2 дня)
- [ ] Прогоны кровли (горизонтальные линии вдоль ската)
- [ ] Вертикальные связи между колоннами
- [ ] Горизонтальные связи по нижнему поясу ферм
- [ ] Раскосы (крестовые связи) в торцевых секциях
- [ ] Всё с правильной изометрической проекцией (30°)

#### Этап 4. Интерактивность (2 дня)
- [ ] Легенда с подписями элементов (что есть что)
- [ ] Подсветка элемента при наведении + тултип с названием
- [ ] Анимация «сборки» — элементы появляются последовательно
- [ ] Возможность поворота камеры (влево/вправо на 15°)

#### Этап 5. Мобильная версия и оптимизация (2 дня)
- [ ] Адаптивный SVG (резиновый вьюбокс)
- [ ] Упрощённая схема на мобильных (меньше деталей)
- [ ] Оптимизация ререндеров (React.memo, useMemo)

### Референс из Excel
Брать реальные схемы раскладки из листов «Склад производство», «Сельхоз», «Гараж, автосервис» — там посекундный расчёт с маркировкой каждого элемента:
- Колонны: К1, К2, К3 (двутавр, сорт)
- Фермы: Ф1, Ф2 (уголки, раскосы)
- Прогоны: П1, П2 (швеллер, шаг)
- Связи: СВ1, СВ2 (уголок, крестовые)
- Раскосы: Р1, Р2

### Технический стек
```
Графика:    SVG (inline) + изометрическая проекция
Логика:     TypeScript — генерация координат из параметров здания
Анимация:   CSS transitions / @keyframes
Тултипы:    CSS абсолютное позиционирование
```

### Результат
Вместо пустой коробки — **читаемая инженерная схема**: видно колонны, видно фермы, видно связи. Клиент понимает что он покупает.

## 3. 🗺️ Карта проектов с объектами

> **Цель:** На карте (Яндекс.Карты) отмечены реальные построенные объекты с фото и описанием.

### Текущее состояние
- ✅ Яндекс.Карты подключены на региональных страницах
- ❌ Нет страницы «Карта проектов» со всеми объектами
- ❌ Нет маркеров построенных объектов

### План реализации

#### Этап 1. Сбор данных (1 день)
- [ ] Подготовить список объектов (адрес, координаты, тип, площадь, год)
- [ ] Собрать фото для каждого объекта (3–5 шт)
- [ ] Примерная структура данных:

```typescript
interface Project {
  id: string;
  title: string;
  type: "warehouse" | "agriculture" | "service" | "other";
  address: string;
  coords: [number, number]; // [lat, lon]
  area: number;     // м²
  year: number;
  description: string;
  photos: string[]; // URL
  features: string[]; // «кран-балка», «утепление», «сэндвич-панели»
}
```

#### Этап 2. Страница карты (2 дня)
- [ ] Новая страница `/projects-map`
- [ ] Яндекс.Карты с кластеризацией маркеров
- [ ] Кастомные иконки маркеров по типу (склад/сельхоз/сервис)
- [ ] Popup при клике: фото, описание, площадь, год, ссылка на подробнее

#### Этап 3. Интеграция (1 день)
- [ ] Добавить ссылку в навигацию «🗺️ Карта проектов»
- [ ] Добавить блок на главной «Построенные объекты» с мини-картой
- [ ] Связать с региональными страницами (фильтр по региону)

#### Этап 4. Фильтрация и поиск (1 день)
- [ ] Фильтр по типу здания
- [ ] Фильтр по площади (диапазон)
- [ ] Поиск по адресу
- [ ] Список объектов рядом с картой

### Технический стек
```
Карта:    @yandex/ymaps3 (React-компонент)
Данные:   Статический JSON (projects-data.ts) или API эндпоинт
Кластеры: Встроенная кластеризация ymaps3
Фото:     Оптимизированные WebP (next/image)
```

### Результат
Страница с картой, где видны все построенные объекты МАКСТИЛ. Клиент может увидеть что строили в его регионе, кликнуть → посмотреть фото, площадь, год постройки.
