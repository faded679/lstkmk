# Промпт для Windsurf — фикс калькулятора цен

## Контекст

У нас сайт lstkmk.ru — строительство зданий из металлоконструкций ЛСТК. На сайте есть калькулятор стоимости (`src/components/Calculator.tsx`). Рядом лежит Excel-файл с инженерными расчётами (Калькулятор.xlsx) — в нём посекундный расчёт каждого элемента металлоконструкции для 26+ конфигураций зданий.

Задача: сверить калькулятор на сайте с данными из Excel и поправить все расхождения.

## Что в Excel

В файле 5 листов:
- **До 300 м2** — 26+ конфигураций складов (от 28 м² до 9720 м²), каждая с полной спецификацией металла
- **Сельхоз** — 1 конфигурация коровника ПСК (3912 м²)
- **Склад производство** — 3 эталонные конфигурации складов (1080, 1440, 2160 м²)
- **Гараж, автосервис** — 1 конфигурация (288 м²)
- **Цены** — цены на материалы (профлист 1100 ₽/м², сэндвич 3300 ₽/м²)

Ключевые строки в каждом листе:
- `Цена на 1 кв. м` — финальная цена за квадратный метр
- `Площадь` или `Площадь здания` — общая площадь этой конфигурации
- `Продажная цена +20%` — итоговая стоимость с наценкой

## Эталонные цены из Excel (верифицированные)

| Тип здания | Площадь | Цена за м² |
|---|---|---|

## Что не так с текущим калькулятором

Текущий код в `Calculator.tsx`:

```typescript
const standardAreas = [100, 300, 600, 1000, 1440, 2160, 3000, 4000];

const priceMatrix = {
  "small-building":  [9500,  7800,  7000,  6500,  6200,  6000,  5800,  5600],
  warehouse:         [7500,  6500,  5900,  5600,  4648,  4957,  4700,  4500],
  agriculture:       [7800,  6800,  6000,  5500,  5100,  4900,  4719,  4600],
  service:           [8500,  7200,  6300,  5800,  5463,  5200,  5000,  4800],
};
```

**Конкретные расхождения:**

1. **Автосервис 288 м²**: сайт выдаёт ~7 278 ₽/м², а Excel говорит 5 463 ₽/м² (разница −25%)
2. **Склад 105 м²**: сайт выдаёт ~7 475 ₽/м², а Excel говорит 9 515–9 727 ₽/м² (разница −21%)
3. **Склад 140 м²**: сайт ~7 300 ₽/м², Excel 7 982–8 753 ₽/м² (−9-17%)
4. **Сельхоз 3912 м²**: сайт ~4 610 ₽/м², Excel 4 719 ₽/м² (−2%)
5. Для складов 1080/1440/2160 — совпадает, это хорошо

**Причина ошибок:**
- Нет опорной точки на 288 м² для гаража
- Мало опорных точек для мелких площадей (<300 м²)
- Сельхоз: после референсной точки цена не должна снижаться

## Что нужно сделать

### 1. Обновить `standardAreas` и `priceMatrix`

Добавить опорные точки, чтобы эталонные цены были точными:

```typescript
const standardAreas = [100, 168, 288, 500, 1080, 1440, 2160, 3000, 4000, 10000];

const priceMatrix: Record<BuildingType, number[]> = {
  "small-building":  [9500,  8600,  8000,  7200,  6500,  6200,  6000,  5800,  5600,  5400],
  warehouse:         [9000,  7500,  6200,  5800,  5520,  4648,  4957,  4500,  4200,  3500],
  agriculture:       [7800,  6800,  6000,  5500,  5100,  4900,  4719,  4719,  4719,  4719],
  service:           [8500,  6500,  5463,  5200,  5000,  4900,  4800,  4700,  4600,  4500],
};
```

### 2. Проверить что интерполяция работает корректно

Функция `getPriceFromMatrix` уже делает линейную интерполяцию. После замены матрицы она автоматически даст правильные значения для эталонных точек.

### 3. Проверить результат

После фикса запустить проверку:

| Тип | Площадь | Должно быть | Статус |
|---|---|---|---|
| service | 288 | 5 463 | ✅ ровно |
| warehouse | 1080 | 5 520 | ✅ ровно |
| warehouse | 1440 | 4 648 | ✅ ровно |
| warehouse | 2160 | 4 957 | ✅ ровно |
| agriculture | 3912 | 4 719 | ✅ ровно |
| warehouse | 105 | ~8 890 | (Excel: 9 515-9 727) |
| service | 200 | ~6 223 | (приемлемо) |

### 4. Коммит

```
fix(calculator): синхронизировать цены с инженерным расчётом Excel

- Добавлен брейкпоинт 288 м² для точного попадания в эталон гаража
- Добавлен брейкпоинт 168 м² для более точных цен на малые склады
- Сельхоз: цена фиксируется на 4719 ₽/м² после 3000 м²
- Мелкие склады (<300 м²): повышены на 10-20% согласно Excel
- Автосервис 288 м²: исправлено с 7278 → 5463 ₽/м² (−25%)
```
