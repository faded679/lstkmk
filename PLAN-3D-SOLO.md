# 🏗️ 3D-конфигуратор зданий — план для одного разработчика

**Референс:** evrazsteelbox.ru/configurator-3d/calc_page/
**Исполнитель:** 1 человек + Windsurf AI
**Цель:** конфигуратор, который считает цену и показывает здание в 3D

---

## ⚠️ Честно upfront: что ты НЕ сделаешь один

В евразовском конфигураторе есть вещи, которые делала команда инженеров-расчётчиков:

| Фича | Почему не потянешь один |
|---|---|
| 5 серий каркасов с расчётом по СП/СНиП | Нужен инженер-конструктор |
| Крановые нагрузки 3.2/5/10т + подкрановые пути | Нужен расчётчик |
| Сейсмика по СП 14.13330 | Нужен расчётчик |
| BIM-модели в GLTF (из Revit/Tekla) | Нужен проектировщик |

> **Что делать:** договорись с директором что ты делаешь визуально-калькуляционную часть, а расчётные формулы берёте из Excel-калькулятора который у вас уже есть. Инженерные данные — не твоя зона.

---

## 🎯 Что ты ДЕЛАЕШЬ (реалистичный скоуп)

1. 3D-модель здания (стены, крыша, колонны, фермы) — крутится мышкой
2. Ползунки габаритов (Ш×Д×В) — модель перестраивается в реальном времени
3. Выбор материалов (сэндвич/профлист, толщина, 6-8 цветов)
4. Цена пересчитывается на лету
5. Окна, ворота, двери — добавил на модель → увидел где стоят
6. Кнопка «Скачать КП» → PDF с картинкой и ценой
7. Отправка заявки в CRM

---

## 🛠️ Стек (то что ты знаешь + 1 новая библиотека)

```
Фреймворк:  Next.js 16 (уже используешь)
3D:         Three.js + React Three Fiber (придётся выучить)
Стили:      Tailwind (уже используешь)
Состояние:  Zustand (проще чем Redux для такого)
PDF:        html2canvas + jsPDF (без сервера, прямо в браузере)
Данные:     Твой Excel-калькулятор → JSON-конфиг с ценами
```

**React Three Fiber** — это Three.js обёрнутый в React-компоненты. Пишешь `<mesh>` как `<div>`. После Next.js вкатиться за 2-3 дня.

### НЕДЕЛЯ 1 — Освоение Three.js

**Задача:** научиться рендерить куб и крутить его мышкой.

- [ ] День 1: Туториал React Three Fiber (https://docs.pmnd.rs)
  - Установить `three`, `@react-three/fiber`, `@react-three/drei`
  - Сделать сцену: пол, свет, куб, OrbitControls (вращение мышкой)
  - Результат: куб крутится на странице Next.js
- [ ] День 1-2: Сделать параллелепипед с размерами
  - `Box` меняет размеры через пропсы (ширина, длина, высота)
  - Три ползунка (Ш×Д×В) → параллелепипед перестраивается
  - Результат: ползунки двигают коробку
- [ ] День 3-4: Текстуры и сетка
  - Наложить текстуру сэндвич-панели на стены (повторяющийся паттерн)
  - Нарисовать сетку с шагом 1м (как у Евраза)
  - Плоскость земли с травой/бетоном
  - Результат: коробка похожа на здание, а не на геометрию
- [ ] День 5: Вертикальная/горизонтальная раскладка панелей
  - Переключатель: горизонтальная ↔ вертикальная
  - Текстура поворачивается на 90°
  - Результат: первый «вау»-момент

### НЕДЕЛЯ 2 — Геометрия здания

**Задача:** коробка превращается в здание с крышей.

- [ ] День 1: Скатная крыша
  - Два фронтона (треугольники на торцах)
  - Два ската (наклонные плоскости)
  - Конёк (линия по верху)
  - Уклон из параметра (5-15°)
- [ ] День 2: Типы зданий
  - Склад/ангар (прямоугольник, двускатная крыша)
  - Сельхоз (то же + вентконёк)
  - Автосервис (ниже, шире)
  - Переключение типа → меняется геометрия
- [ ] День 3-4: Реальные размеры
  - Сетка с метровыми делениями на стенах
  - Подписи Ш×Д×В прямо на модели (3D-текст или спрайты)
  - Ограничения: мин/макс габариты, подсветка красным если выход за пределы
- [ ] День 5: Загрузка Excel-данных
  - Твой `Калькулятор.xlsx` → JSON
  - Цена обновляется при смене габаритов и типа
  - Блок с ценой справа/снизу от 3D

### НЕДЕЛЯ 3 — Каркас: колонны и фермы

**Задача:** внутри коробки появляется структура.

- [ ] День 1-2: Колонны
  - Вертикальные стойки вдоль стен (двутавр визуально)
  - Шаг колонн: 6м для склада, 4.8м для сельхоза
  - Количество колонн = длина / шаг + 1
  - Отрисовка: тонкий `Box` с соотношением как у двутавра
- [ ] День 3-4: Фермы
  - Треугольные фермы поверх колонн (стропильная система)
  - Верхний пояс, нижний пояс, раскосы (зигзаг внутри)
  - Шаг ферм = шаг колонн
  - Цвет: красный/оранжевый для отличия от стен
- [ ] День 5: Связи и прогоны
  - Горизонтальные связи между колоннами (тонкие линии)
  - Вертикальные крестовые связи в торцах
  - Прогоны кровли (линии вдоль ската)

> **Windsurf-промпт на неделю:** «Сделай компонент BuildingFrame который принимает габариты и тип здания, отрисовывает колонны с шагом 4.8 или 6м, треугольные фермы, горизонтальные и вертикальные связи. Используй React Three Fiber. Каждый элемент — отдельный mesh с возможностью подсветки при наведении.»

### НЕДЕЛЯ 4 — Цвета и текстуры

**Задача:** здание можно раскрасить.

- [ ] День 1: 8 базовых цветов RAL
  - Палитра цветов (кнопки-кружочки)
  - RAL 9003 (белый), 9006 (серый), 3009 (красный), 5005 (синий), 6005 (зелёный), 8017 (коричневый), 1015 (бежевый), 9005 (чёрный)
  - При выборе цвета → текстура перекрашивается программно (HSL-сдвиг)
- [ ] День 2: Разные цвета стен и кровли
  - Отдельный селектор для стен и для кровли
  - Цена обновляется (цветные панели чуть дороже)
- [ ] День 3: Материалы стен
  - Сэндвич-панель (текстура с горизонтальными швами)
  - Профлист (текстура с вертикальными волнами)
  - Переключатель меняет текстуру
- [ ] День 4-5: Толщина стен
  - Сэндвич 50/80/100/120/150 мм
  - Визуально: стена чуть толще (увеличение `depth` mesh'а)
  - При 150 мм → цена выше, стена толще

### НЕДЕЛЯ 5 — Окна, ворота, двери

**Задача:** добавить элементы заполнения на модель.

- [ ] День 1: Ворота
  - Прямоугольник на фронтальной стене
  - Стандартные размеры: 3×3, 4×4, 4.2×4.5 м
  - Выбор размера из выпадающего списка
  - Ворота отображаются контуром на текстуре стены
- [ ] День 2: Двери
  - Одностворчатая (1×2.1 м) и двустворчатая (2×2.1 м)
  - Позиция: слева/справа от ворот
- [ ] День 3: Окна
  - Одиночные окна (1.2×1.2, 1.5×1.5, 2×1.5 м)
  - Позиционирование: на боковых стенах
  - Количество: ползунок (0–20)
  - Равномерное распределение по длине
- [ ] День 4: Ленточное остекление
  - Сплошная полоса окон по всей длине стены
  - Высота ленты: 1.2 или 1.8 м
  - Переключатель: одиночные окна ↔ лента
- [ ] День 5: Цена обновляется
  - Каждый элемент добавляет стоимость
  - Ворота 4×4: +85 000 ₽
  - Окно: +12 000 ₽
  - Дверь: +25 000 ₽

> **Как сделать визуально:** окна/ворота — это плоскость другого цвета (стекло = голубой, ворота = тёмно-серый) которая «приклеена» к стене. Без вырезания геометрии — это сложно для одного. Просто рисуем поверх.

### НЕДЕЛЯ 6 — UI: панель управления

**Задача:** всё управление в одной боковой панели.

- [ ] День 1-2: Боковая панель

- [ ] День 3: Ползунки с шагом 10 см

  - Ограничения зависят от типа здания
  - При движении ползунка → 3D перестраивается плавно (debounce 100ms)
- [ ] День 4-5: Сводка цены
  - Каркас: 4 650 000 ₽
  - Стены (сэндвич 100 мм, RAL 9003): 1 850 000 ₽
  - Кровля: 920 000 ₽
  - Ворота 4×4: 85 000 ₽
  - Итого: 7 505 000 ₽
  - Разбивка по категориям, раскрывающиеся строки

### НЕДЕЛЯ 7 — PDF и заявка

**Задача:** результат можно скачать и отправить.

- [ ] День 1-2: Кнопка «Скачать КП»
  - html2canvas делает скриншот 3D-модели
  - jsPDF собирает PDF: логотип, картинка, таблица с ценой, контакты
  - Скачивается в 1 клик, без сервера
- [ ] День 3: Сохранение конфигурации
  - Кнопка «Поделиться» → копирует URL с параметрами
  - `?type=warehouse&w=18&l=60&h=8&walls=sandwich100&color=9003`
  - Открыл ссылку → конфигурация восстановилась
- [ ] День 4: Форма заявки
  - Имя, телефон, email
  - Кнопка «Отправить заявку» → webhook в Telegram/CRM
  - В заявке: все параметры + ссылка на конфигурацию
- [ ] День 5: Кнопка «Позвонить менеджеру»
  - Виджет обратного звонка или просто номер телефона
  - Таймер «менеджер перезвонит через 2 минуты»

### НЕДЕЛЯ 8 — Интерактивность 3D

**Задача:** с моделью можно взаимодействовать.

- [ ] День 1: Подсветка при наведении
  - Навёл на колонну → подсветилась, тултип «Колонна К1, двутавр 30Ш1»
  - Навёл на ферму → «Ферма Ф1, пролёт 18м»
  - Навёл на ворота → «Ворота 4×4м, автоматические»
- [ ] День 2: Клик по элементу → действия
  - Клик по стене → открыть селектор цвета
  - Клик по воротам → открыть выбор размера
- [ ] День 3: Режимы просмотра
  - Кнопки: Каркас | Стены | Всё вместе
  - Каркас: видно только колонны+фермы+связи (стены прозрачные)
  - Стены: только оболочка (каркас скрыт)
- [ ] День 4: Анимация сборки
  - Кнопка «Собрать» → элементы появляются по очереди снизу вверх
  - Фундамент → колонны → фермы → прогоны → стены → кровля
  - Длительность: 3-5 секунд
  - Чистый «вау»-эффект для показа клиенту
- [ ] День 5: Сравнение вариантов
  - Кнопка «Сохранить вариант» → карточка внизу
  - Можно сохранить 2-3 варианта и переключаться между ними
  - Полезно: «сэндвич 100 мм vs 150 мм», «с кранами vs без»

## 🤖 Как использовать Windsurf по максимуму

### Что Windsurf делает хорошо (поручай):
- Компоненты React Three Fiber (пишет меши, текстуры, свет)
- UI-компоненты (панели, ползунки, кнопки)
- Утилиты (конвертация форматов, расчёт координат)
- PDF-генерацию (html2canvas + jsPDF)
- Tailwind-вёрстку

### Что делаешь сам (Windsurf не понимает):
- Инженерные формулы из Excel
- Ценовую матрицу (сверять с калькулятором)
- Цвета RAL и текстуры (утверждать визуально)
- UX-решения (куда кнопку, как расположить панель)
- Финальную приёмку

### Промпт-шаблон для сложных компонентов:
```

Принимает пропсы: [список].
Должен отрисовывать: [описание геометрии].
Использовать: [конкретные материалы/цвета].
Обрабатывать hover и click.
Выдай готовый tsx-файл со всеми импортами.
```

## 💻 Что нужно от сервера

На старте — ничего дополнительного. Все расчёты на клиенте (браузер).

| Компонент | Размещение |
|---|---|
| 3D-модель | Рендерится в браузере (WebGL) |
| Расчёт цены | JavaScript на клиенте |
| PDF | Генерится в браузере (html2canvas + jsPDF) |
| Данные (цены, города) | Статический JSON в репозитории |
| Заявка | Webhook → Telegram бот |

Единственный серверный эндпоинт — приём заявки. Можно через Telegram Bot API (бесплатно) или готовый webhook в amoCRM.
